一、需求分析
1.1 核心功能需求
实时视频采集：调用设备摄像头，将用户所处环境的实时画面连续输入 AI。

实时音频采集：通过麦克风拾取用户语音，支持语音唤醒或按键触发，完成自然对话。

多模态理解：AI 需同时分析视频内容（物体、场景、动作、人物表情等）和语音指令，理解用户意图。

自然语音回应：AI 的应答需通过语音合成（TTS）以自然、流畅的语音反馈给用户。

低延迟交互：从用户说话到收到回应的整体延迟应控制在 3 秒以内，保证对话体验。

【创新点2】个性化物体记忆（零样本增量学习）：允许用户通过语音+框选，现场教 AI 认识新物体。AI 记住用户自定义的名称后，后续再次看到该物体能直接识别并说出用户给定的名称。

【创新点3】主动情境感知与智能提示：当视频画面中出现值得关注的目标或状态变化（如物品遗留、危险动作、可优化操作、备忘事项）时，即便用户未提问，AI 也应主动用语音给出简短、有用的提示。

【创新点4】长期对话记忆与用户画像：系统为每个用户建立长期记忆库，跨会话记住用户偏好、常用物品位置、习惯动作等，并随时间动态演化，实现个性化交互。

【创新点6】可解释性视觉问答：当 AI 回答关于图像的问题时，在视频画面上用高亮框标出推理所依据的区域，并用语音简短说明判断理由。

1.2 非功能需求
准确性：视觉理解需能识别常见物体、人物动作、文字、场景等，准确率不低于 85%（在标准测试集下）。

流畅性：语音识别（ASR）在安静环境下字准确率 >95%；TTS 自然度 MOS 分 >4.0。

成本可控：端云协同设计，尽量减少云端大模型调用次数和计算开销，支持离线基础处理。

隐私安全：视频和音频数据不永久存储，处理完成后即时丢弃；需用户明确授权。

【创新点2+4】个性化数据隐私：用户自定义的物体特征向量、长期记忆数据仅存储在本地，云端未经用户明确同意不得访问；用户可随时查看、导出或删除个性化记忆。

【创新点6】可解释性要求：对于关键识别结果（如物体名称、场景类别），系统应能提供视觉上的证据（高亮区域）和简短文字/语音解释。

【创新点3】主动提示的节制性与隐私：主动提示必须满足多重触发条件（如目标置信度>90%、同一提示30秒内不重复），单次会话主动提示平均每分钟不超过1次。用于判断主动提示的画面分析尽量在端侧完成，仅复杂推理才请求云端，用户可随时关闭主动提示功能。

1.3 约束条件
运行环境：主流浏览器（Chrome/Edge）或跨平台框架（如 Electron），支持 WebRTC 采集媒体流。

AI 能力：可使用开源模型或商业 API，优先考虑混合部署方案。

3.2 端云协同策略（融入创新点）
模块	端侧处理	云端处理
视频流	抽帧（每秒 1~2 帧），裁剪 ROI，降分辨率至 224x224	仅发送筛选后的关键帧，调用视觉大模型
语音	VAD 检测有效语音段，本地回声消除，噪声抑制	仅发送有效语音段进行 ASR 识别
简单指令	本地识别“你好”“再见”“停止”等 5~10 个关键短语，直接触发动作	–
复杂对话	–	拼接视觉+文本上下文，调用 LLM 生成回复
【创新点2】自定义物体识别	本地提取 ROI 特征向量，检索本地向量库；若匹配到用户自定义物体，直接返回名称，不调用云端	仅当本地无匹配且置信度低时，云端识别并返回结果，同时可选地提示用户“是否记住这个物体”
【创新点4】长期记忆	存储结构化记忆（键值对+时间戳），使用轻量 Embedding 进行相似关联；每次对话前加载相关记忆拼接到提示词	仅当用户明确授权且需要复杂推理（如跨多模态关系挖掘）时才访问云端长期记忆
【创新点6】可解释性	若云端返回物体坐标或注意力热力图，端侧在视频画面上绘制高亮框，并使用 TTS 播报简短解释	云端大模型需要支持输出区域坐标（如 GPT-4V 的 region 能力）或生成文字说明
【创新点3】主动提示	端侧持续运行目标检测与规则引擎，检测遗留物、危险姿态、场景变化等；命中规则且置信度达标时，直接用本地 TTS 播报提示；与长期记忆联动，检索相关提醒（如“钱包还在门口”）	仅当本地检测到潜在提示场景但无法精确判断（如“桌上有打开的包裹，不确定是否重要”）时，将加密关键帧上传，由大模型决定是否提醒并返回提示语（频率受限：每分钟 ≤1 次）
四、运营成本控制技巧（融入创新点）
原有策略全部保留，新增以下创新点相关的成本控制：

【创新点2】零成本自定义物体学习：用户教学过程完全本地完成（特征提取 + 向量存储），不产生云端费用。后续识别优先命中本地库，避免重复云端调用。经实测，常见家庭环境下可额外减少 15%~20% 的云端物体识别请求。

【创新点4】长期记忆的端侧裁剪：记忆条目采用 LRU 淘汰机制，保留最近 200 条；云端只同步必要摘要（用户可关闭云端同步），单用户日均额外 token 消耗 < 50。

【创新点6】可解释性低成本实现：高亮坐标数据量极小（每帧 < 200 字节），不影响带宽；解释文本由云端大模型生成时附带，无额外调用成本。且可解释性提升了用户信任，减少了因误识别导致的重复提问，间接降低调用次数。

【创新点3】主动提示的极低成本设计：90%以上的主动提示由本地规则触发，不产生云端费用。仅场景复杂、规则无法覆盖时才走云端，日均额外云端调用量 < 3 次，单用户日均成本增加 < $0.002。提示频次智能抑制（相同事件30分钟内不重复，夜间静默等）进一步降低计算开销。

五、实现的关键技术点
浏览器媒体采集：navigator.mediaDevices.getUserMedia({ video: true, audio: true })

端侧视觉模型：TensorFlow.js + MobileNet 或 ONNX Runtime Web

端侧语音 VAD：@ricky0123/vad-web 库

云端 API：OpenAI GPT-4V 或 Azure OpenAI Service（支持多模态）

流式语音合成：Web Speech API 或云端 TTS 流

成本计量：每次云端请求前计算预估 token 数（图片按 1024 像素边长计为 85 token），在网关层累加

【创新点2】本地向量数据库：使用 lancedb 或 sqlite-vss 存储自定义物体特征（如 ResNet 嵌入），支持近邻检索。

【创新点4】长期记忆存储：IndexedDB 存储 JSON 结构化记忆，并利用 fasttext 或 sentence-transformers 的轻量版本（量化后 4MB）进行语义关联检索。

【创新点6】可解释性渲染：Canvas API 叠加绘制矩形框，配合 Web Audio 或 SpeechSynthesis 输出解释。

【创新点3】主动提示触发器：基于 TensorFlow.js 持续运行的目标检测模型，配合手写规则引擎（如“物体离开视野”“危险姿态检测”）。本地音频打断与优先级管理：用户说话时主动提示排队，紧急安全提示可打断。语音控制开关：“别提醒我”“开启主动提示”由本地关键词识别控制，状态存于 localStorage。

七、潜在风险及应对
隐私泄露：所有视频/音频处理完即焚，服务器不写日志；端侧模型保留在本地；用户可随时清除缓存。对于长期记忆：增加“一键遗忘”功能，本地数据加密存储。

网络抖动导致识别失败：增加重试机制（最多 2 次），并回退显示“网络不佳，请重试”。

大模型幻觉：对物体识别要求 LLM 加上“如果我不确定，请说‘看不清楚’”的提示词约束。对于可解释性：要求模型必须提供坐标证据，若无则回答“我不确定原因”。

【创新点2】用户误教错误物体：提供“撤销最后一次教学”或“忘记该物体”的语音命令，并在界面上显示已学物体列表供管理。

【创新点4】长期记忆导致刻板印象：定期（如每周）提示用户回顾记忆，并允许用户修正；系统自动弱化超过 30 天未使用的记忆。

【创新点3】过度打扰风险：采用渐进式提示策略，新用户默认低频，可通过“多提醒我”逐步增加；提供每日提示次数上限设置。

【创新点3】误报引发信任下降：所有主动提示必须附带高置信度门槛，并在提示语中加入“似乎”“可能需要”等不确定性措辞；允许用户立即反馈“错了”，系统据此调低该类提示的触发阈值。

【创新点3】隐私敏感场景误提示：本地敏感信息过滤清单（如OCR识别到连续数字先静默），主动提示不读出具体敏感字符串，只笼统提醒。
